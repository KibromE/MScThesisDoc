\section{Controller Structure}
\begin{figure}[ht]
    \centering

    \caption{Overall controller structure of the VSR under unbalanced grid voltage conditions}
    \label{fig:controllerStructure}

The control system adopts a cascaded loop structure to separate the fast dynamics of the grid currents
from the slower dynamics of the DC link capacitor energy. Inner Loop (Current Control): A high-bandwidth
PI controller operating at the switching frequency ($f_{sw} = 20 \text{ kHz}$, $T_{inner} = 50 \mu\text{s}$)
regulates the AC currents to track references.Outer Loop (MPC): A Model Predictive Controller operates at
a lower sampling rate ($f_{dc} = 2 \text{ kHz}$, $T_{dc} = 500 \mu\text{s}$) to generate optimal current references
($\mathbf{u}^*$). This multi-rate approach significantly reduces the computational burden of the optimization solver
while maintaining high-fidelity current tracking.\par
\section{Current Controller}
The internal PI current controller's are popular in the industrial application for the reasons, that it is easy to implement
and has low computational burden which is a nesseccity for high bandwidth systems like the active rectifier.\par
The mathematical model developed in equation (\ref{eq:state_space_dq_positive})
and (\ref{eq:state_space_dq_negative}) describes the dynamics of the system
and serves as the basis for the d and q current controller design. The $\omega L$
term presents a coupling effect between the d and q axes, which must
be considered in the controller design.\par

\subsection{Cross Coupling Cancellation}
To design a PI controller for the current control loop, the cross coupling terms must be cancelled out \cite{electronics12112417, sanjuan2010voltage}.
This is done by adding a feed forward term to the controller as shown in Figure \ref{fig:decoupledController}.
The control variable $\mathcal{V}_{d}$ and $\mathcal{V}_{q}$ are defined as in equations (\ref{eq:control_positive})
and (\ref{eq:control_negative}), the system can be decoupled and the cross coupling terms are cancelled out.
\begin{equation}
    \begin{split}
        \mathcal{V}_{d}^{p}    &= -\mathcal{U}_{d}^{p} + e_{d}^{p} + \omega L i_{q}^{p} \\
        \mathcal{V}_{q}^{p}    &= -\mathcal{U}_{q}^{p} + e_{q}^{p} - \omega L i_{d}^{p}
    \end{split}
    \label{eq:control_positive}
\end{equation}
\begin{equation}
    \begin{split}
        \mathcal{V}_{d}^{n}    &= -\mathcal{U}_{d}^{n} + e_{d}^{n} - \omega L i_{q}^{n} \\
        \mathcal{V}_{q}^{n}    &= -\mathcal{U}_{q}^{n} + e_{q}^{n} + \omega L i_{d}^{n}
    \end{split}
    \label{eq:control_negative}
\end{equation}
where $\mathcal{U}_{d}$ and $\mathcal{U}_{q}$ are the new control variables for the decoupled system.
The system can now be represented as in equations (\ref{eq:decoupled_positive}) and (\ref{eq:decoupled_negative}):
\begin{equation}
    \begin{split}
        L \frac{di_{d}^{p}}{dt} &= -R i_{d}^{p} +  \mathcal{U}_{d}^{p}\\
        L \frac{di_{q}^{p}}{dt} &= -R i_{q}^{p} +  \mathcal{U}_{q}^{p}
    \end{split}
    \label{eq:decoupled_positive}
\end{equation}
\begin{equation}
    \begin{split}
        L \frac{di_{d}^{n}}{dt} &= -R i_{d}^{n} +  \mathcal{U}_{d}^{n}\\
        L \frac{di_{q}^{n}}{dt} &= -R i_{q}^{n} +  \mathcal{U}_{q}^{n}
    \end{split}
    \label{eq:decoupled_negative}
\end{equation}
The Laplace transform of the decoupled system $G(s)$ is given in equations (\ref{eq:decoupled_laplace}):
\begin{equation}
    G(s) = \frac{I(s)}{U(s)} = \frac{1}{Ls + R}
    \label{eq:decoupled_laplace}
\end{equation}
\subsection{PI Current Controller Design}
The PI current controller is designed based on decoupled differential equation in equation (\ref{eq:decoupled_laplace}).
Let's define an auxiliary control variable $\mathcal{U}_{d}$ and $\mathcal{U}_{q}$ as in equations
(\ref{eq:aux_control_positive}) and (\ref{eq:aux_control_negative}):
\begin{equation}
    \begin{split}
        \mathcal{U}_{d}^{p}    &= (i_{dref}^{p} - i_{d}^{p}) (k_{p} + \frac{k_{i}}{s})\\
        \mathcal{U}_{q}^{p}    &= (i_{qref}^{p} - i_{q}^{p}) (k_{p} + \frac{k_{i}}{s})
    \end{split}
    \label{eq:aux_control_positive}
\end{equation}
\begin{equation}
    \begin{split}
        \mathcal{U}_{d}^{n}    &= (i_{dref}^{n} - i_{d}^{n}) (k_{p} + \frac{k_{i}}{s}) \\
        \mathcal{U}_{q}^{n}    &= (i_{qref}^{n} - i_{q}^{n}) (k_{p} + \frac{k_{i}}{s})
    \end{split}
    \label{eq:aux_control_negative}
\end{equation}
where $k_{p}$ and $k_{i}$ are the proportional and integral gains of the PI controller respectively.\par
The closed loop transfer function of the system can be derived as in equation (\ref{eq:closed_loop_transfer_function}):
\begin{equation}
    \frac{I(s)}{I_{ref}(s)} = \frac{(k_{p} + \frac{k_{i}}{s}) \frac{1}{Ls + R}}{1 + (k_{p} + \frac{k_{i}}{s}) \frac{1}{Ls + R}}
    = \frac{k_{p} s + k_{i}}{L s^{2} + (R + k_{p}) s + k_{i}}
    \label{eq:closed_loop_transfer_function}
\end{equation}
To achieve a desired closed loop performance, among the various tuning methods, Internal Model Control (IMC)
has been used for PI controller tuning for past decades in systems where an approximate model is available.
This technique allows for adequate tracking and excels in disturbance rejection \cite{Shamsuzzoha2012, artIMCPIDShah}.
The imperfect model should be incorporated as part of the controller design using an explicit model of the plant
\cite{Shamsuzzoha2012}. This tuning provides robustness to the controller, with the control loop remaining stable
even if the parameters change substantially from those used for tuning. This is an interesting feature for the real
implementation of the VSC.\par
The primary disadvantage of IMC PI tuning is that it sets the PI integral part to be equal to
the process time constant. Therefore, if the process had a long time constant, the controller
would have an equally long integral time, resulting in slow recovery from process disturbances.
However, for the VSC application, the process time constant is fast enough that it does not
interfere with the desired controller behavior.\par
The IMC tuning rules for a first order system $P(s)$ is given in equations (\ref{eq:IMC_tuning_kp}) and (\ref{eq:IMC_tuning_ki}):
\begin{equation}
    P(s) = \frac{K}{\tau s + 1}
    \label{eq:first_order_system}
\end{equation}
\begin{equation}
    k_{p} = \frac{\tau}{K \tau_c}
    \label{eq:IMC_tuning_kp}
\end{equation}
\begin{equation}
    k_{i} = \frac{k_p}{\tau}
    \label{eq:IMC_tuning_ki}
\end{equation}
where $\tau_c$ is the desired closed loop time constant and $\tau$ is the process time constant.\par
A smaller $\tau_c$ results in a faster response but reduces the robustness of the controller.
Conversely, a larger $\tau_c$ increases robustness but slows down the response.\par
Thus, to design the PI current controller using IMC tuning rules, the process gain $K$ and time constant $\tau$
must be identified from the decoupled transfer function in equation (\ref{eq:decoupled_laplace}):
\begin{equation}
    P(s) = G(s) = \frac{K}{\tau s + 1}= \frac{\frac{1}{R}}{\frac{L}{R}s + 1}
    \label{eq:transfer_function_identification}
\end{equation}
From equation (\ref{eq:transfer_function_identification}), the process gain $K$ and time constant $\tau$ are identified as:
\begin{equation}
    K = \frac{1}{R}, \quad \tau = \frac{L}{R}
    \label{eq:process_parameters}
\end{equation}
From equations (\ref{eq:IMC_tuning_kp}), (\ref{eq:IMC_tuning_ki}) and (\ref{eq:process_parameters}), the PI controller gains
can be derived as:
\begin{equation}
    k_{p} = \frac{L}{\tau_c}
    \label{eq:final_kp}
\end{equation}
\begin{equation}
    k_{i} = \frac{R}{\tau_c}
    \label{eq:final_ki}
\end{equation}
Once the PI controller gains are determined, they can be implemented in the closed loop transfer function
(\ref{eq:final_closed_loop_transfer_function}) to analyze the system performance obtained from equation
(\ref{eq:closed_loop_transfer_function}) to analyze the system performance.
\begin{equation}
    \begin{split}
        \frac{I(s)}{I_{ref}(s)} &=  \frac{(k_{p} + \frac{k_{i}}{s}) \frac{1}{Ls + R}}{1 + (k_{p} + \frac{k_{i}}{s}) \frac{1}{Ls + R}} \\
        \frac{I(s)}{I_{ref}(s)} &= \frac{1}{\tau_c s + 1}
    \end{split}
    \label{eq:final_closed_loop_transfer_function}
\end{equation}
The bandwidth of the current controller is chosen to be at least ten times smaller than the
switching frequency of the inverter to ensure that the controller can respond quickly enough to
changes in the reference current.
\begin{equation}
    \tau_c = \frac{10}{2 \pi f_{s}}
    \label{eq:time_constant_bandwidth}
\end{equation}
\section{DC Voltage MPC Controller}
The Model Predictive Controller (MPC) is designed to regulate the DC link voltage of the VSR under unbalanced grid conditions.
\begin{figure}[ht]
    \centering
    \includegraphics[scale=0.6]{img/MPCPrediction&ControlHW.png}
    \caption{Illustration of Prediction Horizon ($N_p$) and Control Horizon ($N_c$) in MPC \cite{Kaiser_Kutz_Brunton, currie2014practical}}
    \label{fig:MPC_Horizons}
\end{figure}
The DC voltage controller is formulated to  balance computational efficiency with control stability, the MPC design decouples
the Prediction Horizon ($N_p$) from the Control Horizon ($N_c$). As illustrated in Figure \ref{fig:MPC_Horizons}:
\begin{itemize}
    \item \textbf{Prediction Horizon} ($N_p$): The window over which the controller evaluates the state trajectory (DC Link Energy and actual currents).
          A longer $N_p$ ensures stability by allowing the controller to foresee the long-term integration of energy.
    \item \textbf{Control Horizon} ($N_c$): The number of future control moves the optimizer is free to vary. Beyond $N_c$,
          the control input is constrained to remain constant.
\end{itemize}
Eventhough selecting \textbf{Blocked Horizon}($N_c < N_p$) is preferable for computational efficiency, it has a trade-off with tuning,
control performance and stability. So for high bandwidth control system it is preferred to use \textbf{Full Horizon}($N_c = N_p = N$).\par
The DC voltage controller is designed to regulate the DC link voltage of the VSR to a desired reference value.
The central objective is to regulate the DC link energy ($W = v_{dc}^2$) rather than the voltage directly,
as this linearizes the relationship between power and the state variable as shown in equation (\ref{eq:dc_link_dynamics}).\par
\subsection{Prediction Model}
Discretizing equation (\ref{eq:dc_link_dynamics}) using the Forward Euler method with sampling time $T_s$ of the DC control
loop yields the discrete-time prediction model:
\begin{equation}
    W(k+1) = W(k) + \frac{2 T_s}{C} \left( P_{in}(k) - P_{load}(k) \right)
    \label{eq:dc_link_discrete_model}
\end{equation}
where $P_{in}(k)$ is the instantaneous real power drawn from the grid, and $P_{load}(k)$ is the estimated load disturbance.\par
Under unbalanced grid conditions, the system is modeled using Positive ($dq^p$) and Negative ($dq^n$) sequence components.
The control input vector is defined as the current references:
\begin{equation}
    \mathbf{u}(k) = \begin{bmatrix} i_d^p(k) & i_q^p(k) & i_d^n(k) & i_q^n(k) \end{bmatrix}^T
    \label{eq:control_input_vector}
\end{equation}
The instantaneous power components—Average Real Power ($P_0$), Average Reactive Power ($Q_0$), and Second-Order Ripples
($P_{c2}, P_{s2}$)—are linearly related to the currents via the grid voltage matrix $\mathbf{G}(k)$:
\begin{equation}
    \mathbf{y}(k) = \begin{bmatrix} P_0(k) \\ Q_0(k) \\ P_{c2}(k) \\ P_{s2}(k) \end{bmatrix} = \mathbf{G}(k) \mathbf{u}(k)
    \label{eq:Auxiliary_term_sample}
\end{equation}
Here, $\mathbf{G}(k)$ is a $4 \times 4$ time-varying matrix dependent on the measured grid voltages $e_{dq}^p$ and $e_{dq}^n$ drived in
equation (\ref{eq:power_coefficients}).
\begin{itemize}
    \item Row 1 ($P_0$): Drives the DC link energy state $W$.
    \item Row 2-4 ($Q_0, P_{c2}, P_{s2}$): Auxiliary outputs to be minimized (Regulation Objectives).
\end{itemize}
The decision vector of unknown control inputs, $\mathbf{U}$, is:
\begin{equation}
    \mathbf{U} = [\mathbf{u}(k)^T, \mathbf{u}(k+1)^T, \dots, \mathbf{u}(k+N_c-1)^T]^T
    \label{eq:decision_vector}
\end{equation}
Recall your single-step dynamics in equation (\ref{eq:dc_link_discrete_model}) and expressing it in general format:
\begin{equation}
    W(k+1) = \underbrace{1}_{A} \cdot W(k) + \underbrace{\mathbf{B}}_{B_{sys}} \mathbf{u}(k) + \underbrace{D}_{D_{dist}} P_{load}(k)
\end{equation}
Since $A=1$, the state simply accumulates (integrates) the inputs and disturbances over time.
The future states of the DC energy over the full prediction horizon ($N_p$) are related to this reduced input vector by the modified prediction model:
\begin{equation}
    \mathbf{W}_{pred} = \mathbf{S}_x W(k) + \mathbf{S}_u \mathbf{U} + \mathbf{S}_d P_{load}(k)
    \label{eq:prediction_model_matrix}
\end{equation}
Where the matrices are adapted for the horizon difference:
\begin{itemize}
    \item $\mathbf{S}_x$: The free response vector (size $N_p \times 1$).
          $$\mathbf{S}_x = \begin{bmatrix} 1 \\ 1 \\ \vdots \\ 1 \end{bmatrix}$$
    \item $\mathbf{S}_u$: The forced response matrix (size $N \times 4N$). Crucially, the last column of this matrix accumulates
          the effect of the final control input $\mathbf{u}(k+N_c-1)$ being applied repeatedly for the remainder of the prediction window.
          \begin{equation}
              \mathbf{S}_u = \begin{bmatrix}
                  \mathbf{B}                   & \mathbf{0}                   & \mathbf{0}                   & \cdots & \mathbf{0} \\
                  \mathbf{A}\mathbf{B}         & \mathbf{B}                   & \mathbf{0}                   & \cdots & \mathbf{0} \\
                  \mathbf{A}^2\mathbf{B}       & \mathbf{A}\mathbf{B}         & \mathbf{B}                   & \cdots & \mathbf{0} \\
                  \vdots                       & \vdots                       & \vdots                       & \ddots & \vdots     \\
                  \mathbf{A}^{N_p-1}\mathbf{B} & \mathbf{A}^{N_p-2}\mathbf{B} & \mathbf{A}^{N_p-3}\mathbf{B} & \cdots & \mathbf{B}
              \end{bmatrix}
          \end{equation}
    \item $\mathbf{S}_d$: The disturbance response vector (size $N_p \times 1$).
          $$\mathbf{S}_d = \begin{bmatrix} D \\ 2D \\ 3D \\ \vdots \\ N_p D \end{bmatrix}$$
\end{itemize}
\subsection{Cost Function Formulation}
The MPC solves an optimization problem at each step $k$ to minimize a cost function $J$ over the entire prediction horizon $N_p$.
Even though we only calculate $N_c$ unique inputs, the cost function evaluates the performance of the system up to $N_p$ to ensure
long-term stability.\par
The cost function aims to achieve three objectives:
\begin{itemize}
    \item \textbf{Tracking Accuracy}: Minimize the error between $W$ and $W_{ref}$ over $N_p$ steps.
    \item \textbf{Ripple Suppression}: Minimize power ripples ($P_{c2}, P_{s2}$) and reactive power ($Q$) actively.
    \item \textbf{Control Effort}: Minimize the magnitude of the currents to prevent spikes.
\end{itemize}
The scalar cost function is formulated as:
\begin{equation}
    \begin{split}
        J(\mathbf{U}) &= \sum_{i=1}^{N_p} \left( \underbrace{\lambda_1 (W_{ref} - W(k+i))^2}_{\text{Energy Tracking (Long Term)}} \right) \\
        &+ \sum_{j=0}^{N_c-1} \left( \underbrace{\lambda_2 Q_0(k+j)^2 + \lambda_{3} P_{rip}(k+j)^2}_{\text{Auxiliary Minimization}}
        + \underbrace{\mathbf{u}(k+j)^T \mathbf{R} \mathbf{u}(k+j)}_{\text{Control Effort}} \right)
    \end{split}
    \label{eq:mpc_cost_function}
\end{equation}
Where:
\begin{itemize}
    \item $\lambda_1$: Prioritizes DC voltage tracking stiffness.
    \item $\lambda_2, \lambda_3$: Penalizes reactive power and power ripples. High values here force the controller to use
          Negative Sequence currents to cancel oscillations actively.
    \item $\mathbf{R}$: A regularization matrix to smooth the control action and prevent numerical ill-conditioning.
\end{itemize}
The \textbf{Energy Tracking} term runs up to $N_p$, ensuring that the controller predicts the long-term consequence of its actions.
The \textbf{Auxiliary Minimization} and \textbf{Control Effort} terms are summed up to $N_c-1$, corresponding to the active optimization window.\par
\subsection{Constraints}
The optimization is subject to physical constraints on the converter current rating. By using the reduced control horizon, we enforce
these constraints on the optimized variables directly.
\begin{equation}
    -I_{max} \leq \mathbf{u}(k+j) \leq I_{max}, \quad \forall j \in [0, N_c-1]
    \label{eq:control_input_constraint}
\end{equation}
This imposes "Hard constraints" on the $d$ and $q$ axis currents for the active control moves. Since the inputs beyond $N_c$ are
held constant equal to $\mathbf{u}(k+N_c-1)$, satisfying the constraint for the last active move guarantees satisfaction for the entire horizon.\par
\subsection{Optimization Problem (Quadratic Programming)}
The optimization problem is formulated by substituting the prediction model into the cost function,
the problem is reformulated as a standard Quadratic Programming (QP) problem with box constraints:
\begin{equation}
    \begin{split}
        \min_{\mathbf{U}} \quad \frac{1}{2} \mathbf{U}^T \mathbf{H}_{qp} \mathbf{U} + \mathbf{f}_{qp}^T \mathbf{U} \\
        \text{subject to} \quad \mathbf{LB} \leq \mathbf{U} \leq \mathbf{UB}
    \end{split}
    \label{eq:QP_optimization_problem}
\end{equation}
Where:
\begin{itemize}
    \item \textbf{Decision Vector} $\mathbf{U}$: Has size $N_c \times 4$.
    \item \textbf{Hessian Matrix} ($\mathbf{H}_{qp}$): A positive-definite matrix of size $(4N_c \times 4N_c)$.
          This matrix condenses the cost information from the full $N_p$ horizon into the smaller $N_c$ variable space.
    \item \textbf{Gradient Vector} ($\mathbf{f}_{qp}$): represents the relationship between the current state $W(k)$,
          the reference $W_{ref}$, and the gradient of the cost surface
\end{itemize}
To formulate $\mathbf{H}_{qp}$ and $\mathbf{f}_{qp}$, the cost function in equation (\ref{eq:mpc_cost_function}) should be expressed
in standard linear algebraic form. This involves substituting the prediction model into the cost function.
The standard QP problem minimizes:
\begin{equation}
    J(\mathbf{U}) = \frac{1}{2} \mathbf{U}^T \mathbf{H}_{qp} \mathbf{U} + \mathbf{f}_{qp}^T \mathbf{U} + \text{constant}
    \label{eq:standard_qp_form}
\end{equation}
Instead of writing the cost function in equation (\ref{eq:mpc_cost_function}) in sums, stack the terms in full matrix form here,
\begin{equation}
    J = \underbrace{(\mathbf{W}_{ref} - \mathbf{W}_{pred})^T \mathbf{Q}_{track} (\mathbf{W}_{ref} - \mathbf{W}_{pred})}_{\text{Energy Tracking}}
    + \underbrace{\mathbf{U}^T \mathbf{R}_{reg} \mathbf{U}}_{\text{Control Effort}}
    + \underbrace{\mathbf{Y}_{aux}^T \mathbf{Q}_{aux} \mathbf{Y}_{aux}}_{\text{Auxiliary Minimization}}
    \label{eq:mpc_cost_function_matrix}
\end{equation}
where:
\begin{itemize}
    \item $\mathbf{Q}_{track}$: Diagonal matrix of size $N_p \times N_p$ containing $\lambda_1$.
    \item $\mathbf{R}_{reg}$: Diagonal matrix of size $4N_c \times 4N_c$ containing $\lambda_R$.
    \item $\mathbf{Q}_{aux}$: Diagonal matrix containing $\lambda_2, \lambda_3$ (for Q and Ripples).
\end{itemize}
To Integrate the prediction model in equation (\ref{eq:prediction_model_matrix}),
lets defining a new variable \textbf{Tracking Error Vector} ($\mathbf{E}$) as:
\begin{equation}
    \mathbf{E} = \mathbf{W}_{ref} - (\mathbf{S}_x W_k + \mathbf{S}_d)
    \label{eq:tracking_error_vector}
\end{equation}
Using the prediction model in equation (\ref{eq:prediction_model_matrix}), the $(\mathbf{W}_{ref} - \mathbf{W}_{pred})$
term in the cost function can be replaced as shown in equation (\ref{eq:prediction_error}).
\begin{equation}
    \begin{split}
        (\mathbf{W}_{ref} - \mathbf{W}_{pred}) &= \mathbf{W}_{ref} - \mathbf{S}_x W(k) - \mathbf{S}_u \mathbf{U} - \mathbf{S}_d P_{load}(k) \\
        (\mathbf{W}_{ref} - \mathbf{W}_{pred}) &= \mathbf{E} - \mathbf{S}_u \mathbf{U}
    \end{split}
    \label{eq:prediction_error}
\end{equation}
Substituting equation (\ref{eq:prediction_error}) into the tracking part of the cost function in equation (\ref{eq:mpc_cost_function_matrix}),
\begin{equation}
    \begin{split}
        J_{track} &= (\mathbf{E} - \mathbf{S}_u \mathbf{U})^T \mathbf{Q}_{track} (\mathbf{E} - \mathbf{S}_u \mathbf{U}) \\
        J_{track} &= \mathbf{U}^T \mathbf{S}_u^T \mathbf{Q}_{track} \mathbf{S}_u \mathbf{U} - 2 \mathbf{E}^T \mathbf{Q}_{track} \mathbf{S}_u \mathbf{U} + \mathbf{E}^T \mathbf{Q}_{track} \mathbf{E}
    \end{split}
    \label{eq:tracking_cost_expanded}
\end{equation}
To define the auxiliary output vector $\mathbf{Y}_{aux}$, we can stack the auxiliary outputs over the control horizon and based on equation
(\ref{eq:Auxiliary_term_sample}), the linear relationship becomes a block-diagonal matrix $\mathbf{G}_{big}$:
\begin{equation}
    \begin{split}
        \mathbf{Y}_{aux} &= \begin{bmatrix} \mathbf{y}(k)^T & \mathbf{y}(k+1)^T & \dots & \mathbf{y}(k+N_c-1)^T \end{bmatrix}^T \\
        \mathbf{Y}_{aux} &= \mathbf{G}_{big} \mathbf{U}
    \end{split}
    \label{eq:auxiliary_output_vector}
\end{equation}
Similarly, substituting equation (\ref{eq:auxiliary_output_vector}) into the auxiliary part of the cost function in equation
(\ref{eq:mpc_cost_function_matrix}]):
\begin{equation}
    \begin{split}
        J_{aux} &= \mathbf{Y}_{aux}^T \mathbf{Q}_{aux} \mathbf{Y}_{aux} \\
        J_{aux} &= (\mathbf{G}_{big} \mathbf{U})^T \mathbf{Q}_{aux} (\mathbf{G}_{big} \mathbf{U}) \\
        J_{aux} &= \mathbf{U}^T \mathbf{G}_{big}^T \mathbf{Q}_{aux} \mathbf{G}_{big} \mathbf{U}
    \end{split}
    \label{eq:auxiliary_cost_expanded}
\end{equation}
Integarting equations (\ref{eq:tracking_cost_expanded}) and (\ref{eq:auxiliary_cost_expanded}) into the full cost function in equation
(\ref{eq:mpc_cost_function_matrix}), we can identify the Hessian matrix $\mathbf{H}_{qp}$ and gradient vector $\mathbf{f}_{qp}$ as:
\begin{equation}
    \mathbf{H}_{qp} = 2 \left( \mathbf{S}_u^T \mathbf{Q}_{track} \mathbf{S}_u + \mathbf{G}_{big}^T \mathbf{Q}_{aux} \mathbf{G}_{big} + \mathbf{R}_{reg} \right)
    \label{eq:qp_hessian_matrix}
\end{equation}
\begin{equation}
    \mathbf{f}_{qp} = -2 \mathbf{S}_u^T \mathbf{Q}_{track} \mathbf{E}
    \label{eq:qp_gradient_vector}
\end{equation}
\subsection{Solver Selection}
The selection of an appropriate Quadratic Programming (QP) solver is critical for the real-time feasibility
of the proposed MPC strategy. While the initial validation was performed using MATLAB's standard ({\fontfamily{qcr}\selectfont quadprog})
function with the ({\fontfamily{qcr}\selectfont interior-point-convex}) algorithm, this approach is ill-suited
for embedded implementation due to its heavy reliance on dynamic memory allocation and complex linear algebra
libraries. Consequently, the transition to an embedded-compatible solver required selecting an algorithm capable
of deterministic execution and C-code generation.\par
Based on the comparative evaluations presented in the jMPC Toolbox documentation and
literature \cite{currie2014practical}, Wright's Active Set Method ({\fontfamily{qcr}\selectfont quad\_wright}) was
selected over other common embedded solvers such as Hildreth's Method. While Hildreth's algorithm
({\fontfamily{qcr}\selectfont quad\_hildreth}) is noted for its ease of implementation, established benchmarks
indicate that primal-dual active-set methods like Wright's typically exhibit superior convergence rates for small-to-medium
scale problems with general linear constraints. Therefore, this thesis utilizes the ({\fontfamily{qcr}\selectfont quad\_wright})
solver to ensure the optimal balance between computational efficiency and algorithmic robustness required for the $2 \text{ kHz}$ control loop.\par
\section{Phase Locked Loop (PLL)}
To ensure proper operation of the VSR under unbalanced grid voltage conditions,
a robust phase-locked loop (PLL) is essential for accurate grid voltage phase angle and frequency estimation.
The angle information provided by the PLL is crucial for the transformation of
grid voltages and currents into the synchronous reference frame ($d-q$ frame).
The PLL structure used in this work is illustrated in Figure \ref{fig:PLLstructure}.\par
\begin{figure}[ht]
    \centering
    \includegraphics[scale=0.75]{img/PLL_Block_Diagram.png}
    \caption{Block diagram of the PLL structure \cite{sanjuan2010voltage}}
    \label{fig:PLLstructure}
\end{figure}
The PLL takes the three-phase grid voltages as input, which are first transformed into the $\alpha-\beta$
stationary reference frame using the Clarke transformation. The $\alpha-\beta$ voltages are then converted
into the $d-q$ synchronous reference frame using the Park transformation, which requires an initial estimate
of the grid voltage angle $\theta$. The q-axis voltage component $v_{q}$ is used as the error signal for the PLL,
which is processed by a PI controller to adjust the estimated angle $\theta$.\par
The PLL is designed to minimize the q-axis voltage component $v_{q}$, which ideally should be zero when the estimated
angle $\theta$ matches the actual grid voltage angle. The PI controller in the PLL adjusts the estimated angle based on
the error signal, ensuring that the PLL can track changes in the grid voltage angle and frequency accurately.\par
The PLL is designed to according \cite{sanjuan2010voltage,PLL7467498} and the mathematical model of the PLL can be expressed as:
\begin{equation}
    \begin{split}
        \dot{\omega}  &= \gamma_1 \varepsilon\\
        \dot{\theta} &= \omega + \gamma_2 \varepsilon
    \end{split}
    \label{eq: PLL model}
\end{equation}
where $\varepsilon$ is the PLL error signal of $v_q$, $\omega$ is the estimated angular frequency, $\theta$ is the estimated angle,
and $\gamma_1$, $\gamma_2$ are the PLL PI controller gains ($\gamma_1$ is the integral gain, $\gamma_2$ is the proportional gain).\par
The PI controller gains are designed based on the desired PLL bandwidth $\rho$ and $\hat{E}_g$ as shown in equations
(\ref{eq: PLL gain 1}), (\ref{eq: PLL gain 2}) and (\ref{eq: resultant voltage}):
\begin{equation}
    \gamma_1 = \frac{\rho^2}{\hat{E}_g}
    \label{eq: PLL gain 1}
\end{equation}
\begin{equation}
    \gamma_2 = 2 \frac{\rho}{\hat{E}_g}
    \label{eq: PLL gain 2}
\end{equation}
\begin{equation}
    \hat{E}_g = \sqrt{\hat{E}_d^2 + \hat{E}_q^2}
    \label{eq: resultant voltage}
\end{equation}
As shown in Figure \ref{subfig:c angle estimation bal}, the PLL is able to accurately estimate the grid voltage angle under balanced grid conditions,
with the q-axis voltage component $v_q$ converging to zero and the estimated angle $\theta$ tracking the actual grid voltage angle closely.\par
